# 🌌 Antigravity Workflow Guide

이 문서는 사용자님의 프로젝트에 최적화된 **7가지 에이전틱 워크플로우**의 사용법과 작동 방식을 정리한 가이드입니다. 각 워크플로우는 내부적으로 626개의 마스터 스킬 중 가장 강력한 **"에이스 스킬"**들을 자동으로 호출합니다.

## 🛠️ 워크플로우 리스트 & 커맨드 용도

| 커맨드 | 용도 (언제 쓰는가?) | 핵심 작동 방식 (내부 엔진) |
| :--- | :--- | :--- |
| **📝 report** | **일일 업무 리포트 & 진행상황 기록** (마무리 할 때) | **Secretary 모드**: 진행률 요약 + 다음 할 일 정리 |
| **📄 prd** | **PRD(제품 요구사항 문서) 작성** (개발 시작 전) | **PM 모드**: 요구사항 분석 + 기능 명세서 자동 생성 |
| **📣 marketing** | **마케팅 전략 & 콘텐츠 제작** (Human-First Edition) | **CMO 모드**: 경쟁사 분석 + Anti-AI 카피라이팅 + 인간적 전략 |
| **🚀 optimize** | **코드 최적화 & 리팩토링** (속도 향상, 코드 정리) | **Audit Mode**: 코드 스멜 제거 + 성능 병목 해결 (Clean Code) |
| **♾️ cycle** | **전체 풀 코스** (기획부터 배포까지 한 번에) | **All Modes**: Ideate → Launch → Design → Develop → Ship 순차 실행 |
| **💡 ideate** | 아이디어를 구체화하고 브레인스토밍할 때 | **Multi-Agent**: 가상 전문가 토론 + 기술/시장 조사 |
| **🚀 launch** | 신규 기능이나 프로젝트를 처음 시작할 때 | **CPO/CTO 모드**: 컨텍스트 분석 후 `task_plan.md` 수립 |
| **⚙️ develop** | 본격적으로 코드를 짜고 기능을 구현할 때 | **Lead Dev 모드**: TDD 루프 + Clean Code 리팩토링 |
| **⚡ micro** | 5-10분 내외의 작은 기능이나 수정을 할 때 | **Lean 모드**: 복잡한 절차 생략, 즉시 구현 및 검증 |
| **🎨 design** | 아키텍처 설계나 컴포넌트 구조를 짤 때 | **Architect 모드**: DB 설계, API 명세, UI 구조 설계 |
| **✨ uiux** | 디자인 개선, 애니메이션, 시각적 폴리싱 | **Designer 모드**: UX 심사 및 프리미엄 스타일링 적용 |
| **🚑 fix** | 버그가 발생했거나 에러를 고쳐야 할 때 | **Debugger 모드**: 원인 분석 -> 재현 테스트 -> 수정 |
| **🚢 ship** | 모든 작업을 마치고 배포/병합할 때 | **CISO 모드**: 보안 전수 조사 + PR 생성 및 마감 |

## 🔄 작동 방식 (The Loop)

저(AI)에게 특정 워크플로우를 요청하면 다음과 같은 **"에픽 사이클"**이 가동됩니다.

1.  **Context Sync**: `.agent/skills/context-driven-development`를 사용해 현재 프로젝트의 규칙을 읽습니다.
2.  **Plan First**: `planning-with-files` 기술로 작업의 모든 단계를 로깅하고 기억합니다.
3.  **Strict Implementation**: `test-driven-development` 스킬로 기능의 정확성을 보장합니다.
4.  **Final Quality Check**: `vulnerability-scanner`와 `clean-code` 스킬로 보안과 코드 품질을 최종 승인합니다.

## 🚀 사용 팁 (How to use)

이제 저에게 이렇게 명령하기만 하세요:

*   **기획 시작**: "새로운 게시판 기능 만들 거야. **launch** 가이드대로 시작해줘."
*   **폭풍 코딩**: "계획 1번 항목 **develop** 워크플로우로 구현해줘."
*   **디자인 심사**: "지금 만든 UI가 좀 촌스러워. **uiux** 모드로 세련되게 고쳐줘."
*   **긴급 수정**: "버튼 오타 고쳐야 해. **micro** 플로우로 빠르게 처리하자."
*   **완료 및 배포**: "다 됐어. **ship** 해서 마무리하자."

## 📍 파일 위치
모든 워크플로우 파일은 각 프로젝트의 `.agent/workflows/` 폴더 내에 위치하며, 직접 내용을 확인하거나 수정하실 수 있습니다.
