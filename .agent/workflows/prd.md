# ðŸ“„ PRD Workflow (Product Manager)

Transforms abstract ideas into concrete, development-ready specifications using deep analysis.

## 1. Business & Market Analysis (`startup-business-analyst-market-opportunity`)
- [ ] **Market Scan**: Analyze market opportunity and competitor landscape.
- [ ] **Value Proposition**: Define the unique value proposition and target audience personas.

## 2. Requirement Gathering (`context-driven-development`)
- [ ] **Context Extraction**: Analyze chat history, `ideate` results, and `conductor/product.md`.
- [ ] **User Stories**: Draft agile user stories with acceptance criteria.

## 3. Technical Architecture (`architecture-decision-records`)
- [ ] **System Design**: Outline the high-level architecture and necessary tech stack changes.
- [ ] **Decision Records**: Document key technical decisions (ADRs) if new technologies are introduced.

## 4. Drafting the PRD (`planning-with-files`)
- [ ] **Draft**: Generate `PRD.md` containing:
    - **Problem & Goal**
    - **User Stories & Flow**
    - **Functional Requirements**
    - **Non-functional Requirements (Security, Performance)**
    - **UI/UX Wireframe Descriptions**

## 5. Review & Refine
- [ ] **Critique**: Self-review for logical gaps and ambiguity.
